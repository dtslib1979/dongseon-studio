# PARKSY × Claude in Chrome 실행 워크플로우 가이드

> **문서 성격**: 실제로 손에 잡히는 실행 매뉴얼. "어떻게 명령하고, 어떻게 확인하고, 어떻게 편집하느냐"
> **작성일**: 2026.02.19
> **버전**: v1.0
> **전제**: 갤럭시(Termux) → 집 PC(Chrome) 원격 구조

---

## 0. 물리 구조

```
┌─────────────────────┐          ┌──────────────────────────┐
│  갤럭시 (어디서든)     │          │  집 PC (항상 켜져있음)      │
│                     │          │                          │
│  Termux             │ ──────▶  │  Chrome 브라우저           │
│  └─ Claude Code     │  지시     │  └─ Claude in Chrome 확장  │
│     └─ /chrome      │          │     └─ 6개 플랫폼 탭       │
│                     │          │                          │
│  역할: 명령 센터      │          │  역할: 실행 엔진           │
└─────────────────────┘          └──────────────────────────┘
```

**핵심**: 박씨는 갤럭시만 만진다. PC는 켜놓기만 하면 된다.

## 1. 초기 셋업 (1회)

### 1.1 집 PC 설정

```
[ ] Chrome 설치 + 로그인
[ ] Claude in Chrome 확장 설치 (Chrome Web Store → "Claude")
[ ] 확장 핀 고정 (퍼즐 아이콘 → 핀)
[ ] Claude 계정 로그인

[ ] 6개 플랫폼 로그인 상태 유지:
    [ ] smartplace.naver.com (네이버 플레이스)
    [ ] instagram.com
    [ ] blog.naver.com
    [ ] daangn.com (당근마켓)
    [ ] studio.youtube.com
    [ ] tistory.com

[ ] 사이트 권한 설정:
    위 6개 = 허용
    금융/결제 = 차단

[ ] 권한 모드: "Ask before acting" (초기)
[ ] PC 슬립 방지 설정 (전원 옵션 → 절전 안 함)
[ ] Chrome 시작 시 자동 실행 설정
```

### 1.2 갤럭시 Termux 설정

```bash
# Claude Code 설치 확인
claude --version

# Chrome 연결 테스트
claude --chrome

# 연결 확인
/mcp
# → claude-in-chrome 도구 목록 나오면 성공

# 상시 연결 설정 (선택)
/chrome → "Enabled by default"
```

### 1.3 Record Workflow 녹화 (플랫폼별 1회)

PC에서 직접 한다. Claude in Chrome 사이드 패널 열고:

```
1) 네이버 플레이스 소식 등록 — 수동 1회 녹화
2) 인스타 게시물 업로드 — 수동 1회 녹화
3) 네이버 블로그 글 작성 — 수동 1회 녹화
4) 당근 비즈프로필 소식 — 수동 1회 녹화
5) 유튜브 영상 업로드 — 수동 1회 녹화
6) 티스토리 글 작성 — 수동 1회 녹화
```

이건 한 번만 하면 된다. Claude가 패턴을 학습한다.

## 2. 일상 실행 워크플로우

### 2.1 단건 배포 (가장 많이 쓰는 패턴)

갤럭시 Termux에서:

```bash
# 1) Chrome 연결
claude --chrome

# 2) 명령 입력 (자연어)
```

```
호야당 네이버 플레이스에 소식 올려줘.
제목: "주말 한정 — 딸기 크림빵"
내용: 매주 토일만 만드는 딸기 크림빵. 오전 10시 오픈, 선착순 30개.
사진은 이거 써: [사진 경로 또는 URL]
```

```
# 3) Claude가 플랜 보여줌 (Ask 모드)
#    "네이버 스마트플레이스 접속 → 소식 작성 → 제목 입력 → 본문 입력 → 이미지 업로드 → 발행"
#    → 승인하면 PC Chrome에서 실행

# 4) 완료 확인
#    Claude가 "발행 완료" 알려줌
```

### 2.2 멀티플랫폼 동시 배포

```bash
claude --chrome
```

```
parksy.kr/posts/250219 글을 읽고 아래 순서대로 배포해줘:

1) 네이버 플레이스 — 호야당 소식 (핵심 요약 + 이미지)
2) 인스타 — 게시물 (훅 캡션 + 해시태그 30개)
3) 네이버 블로그 — 전체 본문 네이버 톤 (구분선 + 검색키워드)
4) 당근 — 비즈프로필 소식 + 동네홍보

각 플랫폼 맞춤 변환. 실행 전 전체 플랜 보여줘.
```

Claude가 4개 플랫폼 플랜을 한 번에 보여주고, 승인하면 순차 실행.

## 3. 콘텐츠 편집 워크플로우

### 3.1 원본은 어디서 만드냐

```
갤럭시에서 원본 작성 (parksy.kr에 발행)
    │
    ▼
Termux Claude Code에서 변환 명령
    │
    ▼
Claude가 플랫폼별 톤으로 변환
    │
    ▼
변환 결과 확인 → 수정 필요하면 여기서 수정
    │
    ▼
승인 → PC Chrome에서 각 플랫폼에 발행
```

**편집 포인트**: Claude가 변환한 결과를 Termux에서 확인한다. 마음에 안 들면 수정 지시한다.

```
# 변환 결과가 마음에 안 들 때
"인스타 캡션 다시 써줘. 좀 더 짧게. 훅을 질문형으로."

# 해시태그 수정
"해시태그에서 #맛스타그램 빼고 #안양카페 넣어줘"

# 블로그 톤 수정
"너무 딱딱해. 좀 더 수다 떠는 느낌으로."
```

수정 다 끝나면 "이걸로 발행해" 하면 된다.

### 3.2 이미지/영상은 어떻게 보내냐

**방법 1: parksy.kr에 먼저 올려놓기**
```
"parksy.kr/images/250219-bread.jpg 이거 써"
→ Claude가 URL에서 이미지 가져와서 각 플랫폼에 업로드
```

**방법 2: GitHub 레포에 올려놓기**
```
"github.com/[레포]/assets/bread.jpg 써"
```

**방법 3: PC 로컬 경로**
```
"C:\Users\parksy\Pictures\bread.jpg 써"
→ PC Chrome에서 로컬 파일 접근
```

**방법 4: 갤럭시에서 직접 (제한적)**
```
Termux에서 이미지 경로 지정 → Claude Code가 PC로 전달
```

**추천**: parksy.kr 또는 GitHub에 이미지를 먼저 올려놓고 URL로 참조하는 게 가장 깔끔하다. 원본 관리도 되고.

## 4. 일일 루틴 설계

### 4.1 최소 루틴 (하루 10분)

```
아침 (출근 전 또는 매장 오픈 전):

1) 갤럭시 Termux 열기
2) claude --chrome
3) 오늘 배포할 콘텐츠 명령:

"호야당 오늘 소식:
- 플레이스: '오늘의 빵 — 소금빵, 크림치즈빵, 앙버터'
- 인스타: 빵 사진 + 캡션 (parksy.kr/images/250219.jpg)
- 당근: 동네홍보 '호야당 오늘 갓 구운 빵 나왔습니다'
3개 동시에. 플랜 보여줘."

4) 플랜 확인 → 승인
5) 완료 확인
6) Termux 닫기
```

### 4.2 주간 루틴 (추가 30분)

```
주 1회 (주말):

1) 네이버 블로그 — 주간 콘텐츠 발행 (parksy.kr 원본 기반)
2) 유튜브 — 영상 업로드 (촬영해둔 것)
3) 티스토리 — 블로그 글 구글 SEO 버전
```

## 6. 트러블슈팅

### 6.1 "PC Chrome 연결 안 됨"

```
원인: PC가 꺼졌거나 Chrome이 꺼졌거나 확장이 비활성화
해결:
1) PC 전원 확인 (원격 데스크톱으로 확인 가능하면)
2) Chrome 자동 시작 설정 확인
3) 확장 핀 고정 확인
4) Termux에서 다시: /chrome
```

### 6.2 "플랫폼 로그인 풀림"

```
원인: 쿠키 만료 또는 플랫폼이 세션 강제 종료
해결:
1) Claude가 "로그인 필요" 알림 → PC에서 직접 로그인
2) "로그인 상태 유지" 체크 확인
3) 네이버는 2주마다 풀리는 경향 → 주기적 확인
```

### 6.3 "CAPTCHA 뜸"

```
원인: 플랫폼이 자동화 의심
해결:
1) Claude가 멈추고 "CAPTCHA 감지" 알림
2) PC에서 직접 CAPTCHA 해결
3) 이후 Claude가 이어서 진행
4) 반복되면 업로드 간격 늘리기
```

### 6.4 "인스타 계정 잠금"

```
원인: 봇 탐지 (인스타 S급 적대)
해결:
1) 즉시 중단
2) 인스타 앱에서 "내 계정입니다" 인증
3) 24-48시간 대기
4) 재개 시 업로드 간격 최소 2시간 이상
5) Ask 모드로 전환 (자율 실행 끄기)
```

### 6.5 "변환 톤이 마음에 안 듦"

```
해결: Termux에서 바로 수정 지시
"인스타 캡션 다시 써. 지금 너무 광고 같아. 동네 친구한테 말하듯이."
"블로그 제목 바꿔. '수제빵 맛집' 넣어줘."
"해시태그에서 #먹방 빼고 #안양맛집거리 넣어."
→ 수정 확인 후 "이걸로 발행해"
```

## 7. 프롬프트 치트시트 (Termux 즉시 복붙용)

### 원스톱 전체 배포
