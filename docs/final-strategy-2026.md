## 1.1 정체성 재정의

### 기존 정체성

```
"비즈니스의 장면을 그리는 디지털 스튜디오"
→ 웹 제작사의 예쁜 포장
→ 결국 고객 요청에 맞추는 구조
→ 확장 = 인력 채용
→ 끝 = 유지보수 지옥
```

### 새로운 정체성

```
┌────────────────────────────────────────────────────────────┐
│                                                            │
│   DONGSEON = AI 시대 1인 사업자 무기 공장                  │
│                                                            │
│   ──────────────────────────────────────────────────────── │
│                                                            │
│   우리는:                                                  │
│   • 웹사이트를 만들지 않는다 → 시스템을 복제한다          │
│   • 고객에게 맞추지 않는다 → 고객이 우리에게 맞춘다       │
│   • 유지보수하지 않는다 → 버전 업그레이드한다             │
│   • 외주를 받지 않는다 → 무기를 분양한다                  │
│                                                            │
│   ──────────────────────────────────────────────────────── │
│                                                            │
│   핵심 차별화:                                             │
│   "우리가 쓰는 것 = 우리가 파는 것"                        │
│   "증명된 시스템만 판다"                                   │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

## 1.2 제1원칙: 맞춰주지 않는다

### 원칙 선언

```
┌────────────────────────────────────────────────────────────┐
│                                                            │
│   DONGSEON Studio 제1원칙                                  │
│                                                            │
│   "이건 맞춰주는 서비스가 아닙니다.                       │
│    이 방식으로 살아갈 사람만 오세요."                     │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

### 우리가 하는 것 vs 하지 않는 것

| 우리가 하는 것 | 우리가 하지 않는 것 |
|---------------|-------------------|
| ✅ 검증된 시스템 제공 | ❌ 고객별 커스터마이징 |
| ✅ 표준화된 워크플로우 | ❌ "우리 업종은 특이해서" 대응 |
| ✅ 버전 업그레이드 | ❌ 개별 요청 반영 |
| ✅ 커뮤니티 접근권 | ❌ 1:1 밀착 지원 |
| ✅ 교육 콘텐츠 | ❌ 대신 운영해주기 |

### 이 원칙이 만드는 구조

```
외주 지옥 모델:                    무기 공장 모델:
─────────────────                  ─────────────────
고객 10명 = 10가지 버전            고객 100명 = 1가지 버전
매출 ↑ = 일 ↑                      매출 ↑ = 일 동일
고객 많으면 = 죽음                  고객 많으면 = 데이터
확장 = 인력 채용                    확장 = 복제
유지보수 = 끝없는 요청              유지보수 = 버그 수정
```

### 고객 선별 기준

```
✅ 맞는 고객:
├── 시스템을 받아들일 준비가 된 사람
├── 스스로 학습하고 실행할 의지가 있는 사람
├── "왜 이렇게 하죠?" 대신 "어떻게 하면 되죠?" 묻는 사람
└── 우리 방식이 맞다고 느끼는 사람

❌ 안 맞는 고객:
├── "우리 업종은 좀 특이해서요"
├── "이거 이렇게 바꿔주실 수 있나요?"
├── "다 해주시는 거죠?"
└── 자기 방식을 고집하는 사람
```

## 1.3 외주 vs 소프트웨어 비즈니스

### 문법의 차이

```
┌────────────────────────────────────────────────────────────┐
│                                                            │
│   웹 제작 문법 (외주):                                     │
│   ────────────────────                                     │
│   "고객님, 어떻게 해드릴까요?"                             │
│   "네, 수정해드리겠습니다"                                 │
│   "이 부분은 추가 비용이..."                               │
│                                                            │
│   → 시간을 판다                                            │
│   → 고객이 갑                                              │
│   → 확장 불가능                                            │
│                                                            │
│   ──────────────────────────────────────────────────────── │
│                                                            │
│   소프트웨어 비즈니스 문법:                                │
│   ────────────────────────                                 │
│   "이게 우리 시스템입니다"                                 │
│   "이 방식으로 쓰시면 됩니다"                              │
│   "다음 버전에서 개선됩니다"                               │
│                                                            │
│   → 시스템을 복제한다                                      │
│   → 우리가 표준                                            │
│   → 무한 확장 가능                                         │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

### Notion, Slack, Figma의 공통점

```
"우리 업종에 맞게 바꿔주세요" → 안 바꿔줌
"이 기능 추가해주세요" → 로드맵에 넣을게요
"급해요" → 다음 버전 기다리세요

그래도 고객이 쓴다. 왜?
→ 시스템이 좋으니까.
→ 대안이 없으니까.
→ 이미 익숙해졌으니까.

DONGSEON도 이 위치를 노린다.
```

### 패키지 구성

| 패키지 | 포함 내용 | 타겟 |
|--------|----------|------|
| **소상공인** | 매장 소개 + 일상 콘텐츠 시스템 | 카페, 식당, 샵 |
| **부동산** | 매물 소개 + 시장 분석 시스템 | 공인중개사 |
| **1인 사업자** | 전문성 콘텐츠 시스템 | 컨설턴트, 강사 |

### 제1원칙 적용

```
우리가 하는 것:
✅ 표준 칠판 템플릿 제공
✅ 워크플로우 교육 (1회)
✅ 시스템 버전 업그레이드

우리가 하지 않는 것:
❌ 업종별 커스터마이징
❌ 콘텐츠 대신 제작
❌ 채널 대신 운영
❌ "우리 업종은 특이해서" 대응
```

### 수익 모델

```
일회성 납품비: 100-300만원
├── 웹사이트 구축
├── 칠판 시스템 세팅
├── 워크플로우 교육
└── 템플릿 제공

유지: 없음 (고객이 스스로 운영)
업그레이드: 연 1회 버전업 (선택)
```

## 3.3 Lane 2: 자동화 솔루션

### 개요

```
┌────────────────────────────────────────────────────────────┐
│                  LANE 2: 자동화 솔루션                      │
├────────────────────────────────────────────────────────────┤
│                                                            │
│   슬로건: "반복은 기계가, 창의는 당신이"                   │
│                                                            │
│   Core IP: 클로드앤크롬 GUI 자동화                         │
│                                                            │
│   자동화 가능 영역:                                        │
│   ├── 예약 → 자동 확인 문자                               │
│   ├── 문의 → 자동 1차 응대                                │
│   ├── 데이터 수집 → 자동 정리                             │
│   ├── SNS 포스팅 → 예약 발행                              │
│   ├── 재고 확인 → 자동 알림                               │
│   └── 리포트 생성 → 자동 작성                             │
│                                                            │
│   결과:                                                    │
│   하루 2-3시간 반복 업무 → 0시간                          │
│   그 시간에 고객은 더 중요한 일을 한다                    │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

### 제공 방식

```
Step 1: 진단 (무료/유료)
├── 현재 업무 프로세스 분석
├── 자동화 가능 영역 식별
└── 예상 효과 산출

Step 2: 구축
├── 자동화 시스템 구축
├── 테스트 및 검증
└── 인수인계

Step 3: 운영 (고객이 직접)
├── 시스템 사용
├── 문제 시 문서 참조
└── 버전업 시 업그레이드
```

### 제1원칙 적용

```
우리가 하는 것:
✅ 표준 자동화 패턴 적용
```
