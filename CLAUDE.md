# DONGSEON Studio 정관

> 이 문서는 DONGSEON Studio의 정체성, 구조, 운영 원칙을 정의하는 기준 문서다.

---

## 0. 한 줄 정의

**동선(DONGSEON)**은 비즈니스의 장면을 그리는 디지털 스튜디오다.

> 우리는 페이지를 만들지 않는다. 사람을 움직이게 되는 **선(Line)**을 그린다.

---

## 1. 스튜디오 정체성

### 이름
- 한글: 동선
- 영문: DONGSEON
- 해석: Dynamic Line

### 슬로건
- 동선 – 사람이 움직이게 되는 구조
- 동선 – 장사의 장면을 그리다
- 동선 – We Draw How People Move
- 동선 – Business, Drawn

### 핵심 철학
```
코드 = 붓
레이아웃 = 구도
UX = 시선 이동
페이지 = 장면
사이트 = 무대
```

동선은 '웹 제작사'가 아니라 **비즈니스를 그리는 스튜디오**다.

### DONGSEON 관할 범위

```
✅ 웹 제작/납품
✅ AI 연동 (문의, 예약, 안내, 자동화)
✅ LLM 운영
✅ 서버 PC 운영
✅ 온톨로지 구축
✅ 디지털 패키지 (소상공인, 부동산, 1인사업자)
✅ parksy-image, parksy-audio 등 디지털 스튜디오
✅ 물성화되지 않은 모든 IT적 시도
```

## 3. 역할 분리

### 박씨 (설계자)

| 역할 | 내용 |
|------|------|
| 세계관 | 구조, 언어, 철학 설계 |
| 패키지 설계 | 업종별 패키지 구조화 |
| 정의 | "이 업은 이런 생물이다" |
| IP 생성 | 모든 설계/구조/코드 = 박씨 귀속 |

### 동선 (운영자)

| 역할 | 내용 |
|------|------|
| 물리적 거점 | 사무실, 서버 PC |
| 현장 운영 | 세일즈, 납품, 유지보수 |
| 고객 관리 | 기존 고객 풀 활용 |
| 리스크 감당 | 현실 비즈니스 리스크 |

### 핵심 원칙

> **구조는 박씨가 만들고, 현실은 동선이 굴린다.**

## 4. 파트너십 구조

### 관계 정의

```
❌ 투자자-피투자자
❌ 갑-을 (외주)
❌ 고용-피고용

⭕ 친구
⭕ 상생 파트너
⭕ 같이 크는 구조
```

### 수익 구조

| 항목 | 내용 |
|------|------|
| 형태 | 건별 용역비 분배 |
| 비율 | 프로젝트별 협의 (박씨 30~70%) |
| 정산 | 명확한 금액, 분쟁 소지 없음 |
| IP | 모든 설계/구조 = 박씨 귀속 |

### 브랜치와의 차이

| 구분 | 브랜치 (5개) | DONGSEON |
|------|-------------|----------|
| 관계 | 육성 (일방향) | 상생 (양방향) |
| 대상 | 모르는 사람도 포함 | 친구 |
| 수익 | 각자 | 분배 |
| 성장 | 키워주는 것 | 같이 크는 것 |

## 5. 제품 형태 (패키지)

동선 스튜디오는 '페이지'를 팔지 않는다. **시스템**을 판다.

### 패키지 목록

1. **소상공인 패키지 v1**
2. **부동산 패키지 v1**
3. **1인 사업자 자동화 패키지 v1**

### 각 패키지 구성

```
1. 브랜드 서사
2. 행동 동선 설계
3. 웹 장면 연출
4. AI 연동 (문의, 예약, 안내, 자동화)
5. 운영 매뉴얼
```

---

## 6. 물리적 거점

| 자산 | 역할 |
|------|------|
| 동선 사무실 | 현실 앵커, 고객 접점 |
| 서버 PC | 박씨 사고의 물리화, LLM/연산 |
| LLM | 사고 증식 엔진 |

> 그 공간은 더 이상 '1인 IT 사무실'이 아니라, **동선 스튜디오의 첫 기지**가 된다.

---

## 7. 목표

```
❌ 웹 제작사가 아니다.
❌ 마케팅 대행사가 아니다.
❌ 플랫폼도 아니다.
```

동선은:

> **비즈니스의 장면을 그리는 작은 문명 공장이다.**

사람이 멈추지 않는 길.
설득하지 않아도 움직이게 되는 선.

**그게 동선이다.**
